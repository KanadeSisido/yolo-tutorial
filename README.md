# YOLOを使ってオブジェクトを検出する

機械学習を用いたオブジェクト検出を行う例です。
このリポジトリでは、機械学習ライブラリ「ultralytics」で物体認識モデルYOLOv11nを動かし、手持ちのPCのWebカメラで物体認識を行います。

## 事前準備
### PC環境にインストールしてほしいもの
uv (https://docs.astral.sh/uv/#installation)

→ Pythonのパッケージマネージャです。以降のPython開発（ロボット本番環境以外）では、基本的にuvを使いましょう。

windowsの場合は以下のコマンドをPowerShellで実行すれば一発でインストールできます。
```bash
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### 環境設定
0. はじめに、このリポジトリをcloneします。
```bash
git clone https://github.com/KanadeSisido/yolo-tutorial.git
```
1. cloneすると、current directory配下に`yolo-tutorial`というフォルダができます。
最初は、`ls`コマンドで、そのようなフォルダがあるか調べてください。
```bash
ls
```
見つけた`yolo-tutorial`フォルダに入るには`cd`コマンドを使います。
```bash
cd yolo-tutorial
```

2. この場で開発環境を整えましょう。以下のコマンドを実行します。
```bash
uv init
```

3. 以下のコマンドを実行し、今回のプロジェクトで使うライブラリをインストールします。このように、プロジェクトが使うライブラリなどのことを「依存関係（があるライブラリ）」と呼ぶことがあります。 
```bash
uv add opencv-python ultralytics
```
4. 早速、PCにWebカメラを繋いで、以下のコマンドを実行してください。
```bash
uv run video_once.py
```
※カメラが起動し、画面上にカメラ映像が映ります。

## やってみよう
最初に、`video_once.py`のコードを読んでみましょう。
ビデオカメラの映像を一回だけ取得し、それを表示するコードが書かれていて、カメラアプリのように使えます。

ロボコンでは、常にカメラを回し続けることが多いので、次は、カメラのリアルタイム映像を表示するコードを試しましょう。

アイデアは簡単で、`cap.read`（カメラ映像取得）関数と`cv2.imshow`（表示）関数をwhileループ内で無限に呼び出せば良さそうです。
`video.py`を読んでみましょう。まさに今述べたことが実装されています。

実際に動かしましょう。
```bash
uv run video.py
```

カメラ映像がリアルタイムで表示されると思います。このリポジトリのゴールは、この映像に対して機械学習モデルで物体認識することです。

早速コードを見てみましょう。
`yolo.py`は、インターネット上にあるバスの画像に対して推論するコードです。ここで用いている`yolo11n.pt`とは、モデルの種類の名前です。

`n`はもっとも軽量なモデルで、より推論速度が遅い`yolo11s.pt`や`yolo11x.pt`もあります。速度が遅いほど正確な出力を出す傾向がみられます。

機械学習モデルは、通常、モデルを学習する必要があります。大量の画像と、画像中のどこに検出対象があるかを記載するアノテーションデータが必要で、モデルの学習にも強い計算資源が必要です。

今回は、このように時間がかかる学習をせずに、すでにライブラリ提供元が学習してくれたモデル「Pretrainedモデル」を使います。

Pretrainedモデルを使うには特に難しいことはせず、`YOLO('yolo11n.pt')`と書けば自動でインターネット経由でダウンロードしてくれます。

今回のPretrainedモデルは「COCO 2017データセット」で学習されていて、人間（Person）やボトル（Bottle）、バス（Bus）などを検出できます。

もし、自分で決めたものを認識させたい！という場合は、いくつか方法があり、
- まっさらなモデルに自前の画像で学習させる
- すでに学習されたPretrainedモデルに、自前の画像をFine-Tuning（ファインチューニング）する


があります。一般的に、PretrainedモデルにFine-Tuningする後者の方法のほうが性能が上がりやすいとされることが多いです。

> [!NOTE]
> 発展：なぜPretrainedモデルにFine-Tuningするほうが性能が上がりやすいのでしょうか？

実際にYOLOによる推論を動かしてみましょう。
```bash
uv run yolo.py
```

バスの画像に対して推論を行えました。画面にバスの画像（パディングボックスが表示されている）が出ればOKです。

これをリアルタイムのWebカメラ映像に適用することを考えましょう。

さきほどの`video.py`のループの中に、`model`で推論し、画面に表示する処理を書くとうまくいきそうです。
その実装を行ったコードが`yolo_webcam.py`です。

実際に動かしてみましょう。
```bash
uv run yolo_webcam.py
```
すると、画面に映像が表示され、リアルタイム映像の物体認識ができます。やったね！

### ロボットでどう使うか

ロボットでは、画面上のどの座標に物体があるかをピクセル単位でほしいものです。もっと言えばメートル単位で欲しいものです。

まずは、物体認識のパディングボックスのx,y座標を取ることを考えます。

`yolo_getValue.py`は、パディングボックスなどの情報の中身を見る例が含まれています。今書かれているコードは、結果の一部の中身を分析しているコードですが、気になる要素をprintし、覗いてみましょう。

※何度も実行しても推論が回らないように、結果をファイルで一時保存しています。繰り返し実行しても大丈夫です。
